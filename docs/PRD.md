# 產品需求文件：AI 聊天機器人 (AI Chatbot)

## 1. 產品概述
開發一個具備前後端架構的 AI 聊天機器人，目標是超越基本的單次對話，提供包含多聊天室狀態管理、對話歷史回顧、檔案上傳分析、回答控制、個人化記憶以及外部工具整合的全方位進階對話體驗。這將提供使用者一個功能完善且貼近真實應用場景的 AI 協作工具。

## 2. 目標用戶
- 尋求 AI 輔助處理日常任務的使用者。
- 需要多任務並行處理與管理不同對話紀錄的工作者。
- 期望 AI 能夠記住個人偏好以提供持續性個人化體驗的進階使用者。

## 3. 核心功能

### 對話狀態管理 (Session Management)
- **功能描述**：支援建立與維護多個獨立的對話工作階段（session），維持各自的上下文關聯。
- **使用場景**：使用者能同時開啟多個聊天視窗，一個處理程式碼開發，另一個處理出國旅遊規劃，兩者上下文不互相干擾。
- **預期結果**：系統能夠精確區分不同 session 的上下文並維持對談連續性。

### 訊息系統 (Message System)
- **功能描述**：建立結構化的訊息格式，包含角色 (role)、訊息內容 (content) 以及精確的時間戳記 (timestamp)。
- **使用場景**：在聊天介面上正確顯示每一則訊息的發送者與時間，後端也能藉此維持準確的對話時序。
- **預期結果**：前後端統一訊息格式，提供清晰且正確的對話時間線。

### 對話歷史管理 (Chat History Management)
- **功能描述**：記錄所有過去的 session，並提供介面讓使用者可以查看並切換至過去的對話紀錄。
- **使用場景**：使用者想繼續上週與 AI 未完成的英文對話練習。
- **預期結果**：使用者能輕易找到歷史的 session 並無縫接續先前的討論。

### 上傳圖片或文件 (File Upload)
- **功能描述**：支援使用者上傳圖片或其他文件格式，作為對談的輸入背景資料。
- **使用場景**：使用者上傳一張財務報表截圖並詢問 AI 該報表中的異常數值部分。
- **預期結果**：系統能夠處理檔案上傳並正確傳遞給模型，讓 AI 能根據檔案內容提供解答。

### 回答控制 (Response Control)
- **功能描述**：在 AI 回應過後提供「重新生成（Regenerate）」的選項，或在生成中提供「中止回應（Stop）」的控制選項。
- **使用場景**：當使用者意識到 AI 生成的內容方向錯誤時，立即中止以節省時間；或對解答不滿意要求重新生成一個新的版本。
- **預期結果**：強化使用者對對話流程與生成結果的掌控度。

### 記憶機制 (Memory Mechanism)
- **功能描述**：儲存使用者的長期偏好設定或習慣，並能夠跨對話 (cross-session) 實踐這些被提取出的記憶。
- **使用場景**：使用者曾指明「請以繁體中文且輕鬆的口吻回答」，即使開啟全新的對話框，AI 依然記得並遵守該設定。
- **預期結果**：提供高度個人化的跨 session 一致性體驗。

### 工具整合 (Tool Integration)
- **功能描述**：串接外部 API 等工具，使 AI 模型不僅能基於內部知識庫回答，還能主動調用工具獲取資訊或採取行動。
- **使用場景**：使用者詢問「目前的比特幣即時價格是多少？」，AI 辨識需求並自主呼叫市場報價 API 獲取即時數據後回報。
- **預期結果**：突破靜態知識限制，賦予 AI 獲取外部精準與動態資訊的能力。

## 4. 非功能需求
- **技術架構**：前端使用 HTML 等基礎技術開發介面，後端指定使用架構單純且高效能的 FastAPI，資料庫固定採用輕量化的 SQLite。
- **效能需求**：處理對話請求需具備低延遲特性，應盡可能支援回覆介串流輸出 (Streaming) 以優化使用者等待體感。
- **使用者體驗**：聊天介面需友善且直覺，歷史對話列與各項進階功能按鈕必須清楚易見。

## 5. 成功指標
- 所有前述七項核心功能皆能順利連動執行且系統穩定無嚴重報錯。
- 前視窗介面操作與 FastAPI 後端連線正常，且資料能順利保存與讀取自 SQLite 資料庫。
- 順利產出手動測試清單，並通過系統整合測試。

## 6. 實施路線圖
- **階段 1：需求與架構設計**（PRD、架構圖、資料庫 Schema 文件化）。
- **階段 2：基礎服務建置**（架設 FastAPI、設定 SQLite 架構與基本連線）。
- **階段 3：基礎對話系統**（完成多 session 切換與文字歷史儲存讀取）。
- **階段 4：進階對話功能實作**（開發文件上傳、中止與再次生成機制）。
- **階段 5：智能補強實作**（整合記憶機制跨 session 同步與設計外部工具 API 串接）。
- **階段 6：測試部署清點**（執行手動功能清單測試，截圖並完成專案自動提交）。
