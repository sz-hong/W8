## 💡 核心能力

- **需求分析**：從用戶描述中提取核心需求和痛點
- **結構化輸出**：生成符合業界標準的 PRD 格式
- **場景化設計**：設計符合實際使用場景的產品功能
- **迭代優化**：根據反饋持續改進需求文件

## 🎯 適用場景

- 想要將想法轉化為產品需求
- 需要為新產品或功能撰寫 PRD
- 需要將模糊需求轉化為清晰規格
- 需要為開發團隊提供明確指引

## 📝 使用範例

### 範例 1：建立新產品 PRD

**用戶輸入**：
> 我想建立一個線上學習平台，提供課程購買和學習功能

**技能輸出**：
```markdown
# 產品需求文件：線上學習平台

## 1. 產品概述
建立一個提供線上課程購買和學習功能的平台，讓用戶可以隨時隨地學習。

## 2. 目標用戶
- 學生
- 在職人士
- 終身學習者

## 3. 核心功能
- 課程瀏覽和搜索
- 課程購買和支付
- 課程學習和進度追蹤
- 用戶評價和反饋

## 4. 非功能需求
- 響應式設計
- 安全的支付處理
- 穩定的學習體驗

## 5. 成功指標
- 註冊用戶數
- 課程購買率
- 用戶滿意度
```

### 範例 2：優化現有產品功能

**用戶輸入**：
> 我想改進現有產品的註冊流程，讓它更簡單

**技能輸出**：
```markdown
# 產品需求文件：優化註冊流程

## 1. 產品概述
簡化現有產品的註冊流程，減少用戶流失率。

## 2. 問題描述
當前註冊流程過長，需要填寫太多信息，導致用戶放棄註冊。

## 3. 解決方案
- 減少必填字段
- 增加社交媒體登錄選項
- 提供即時驗證反饋

## 4. 預期效果
- 註冊完成率提升 20%
- 用戶流失率降低 15%
- 提升用戶體驗

## 5. 實施步驟
1. 收集用戶反饋
2. 設計新的註冊流程
3. 進行 A/B 測試
4. 全面上線
```

## 🛠️ 使用指南

### 最佳實踐

1. **提供清晰的背景信息**：
   - 說明產品的目標和願景
   - 描述目標用戶和他們的痛點

2. **明確需求範圍**：
   - 說明哪些功能是必須的
   - 哪些功能是可選的
   - 哪些功能是未來的

3. **提供具體細節**：
   - 描述每個功能的具體行為
   - 提供使用場景示例
   - 說明預期結果

4. **迭代優化**：
   - 根據反饋修改需求
   - 逐步完善需求文件
   - 確保需求清晰可執行

### 錯誤示例

**錯誤輸入**：
> 我想做一個 app

**原因**：
- 需求太模糊
- 沒有提供任何細節
- 不清楚產品目標

**正確輸入**：
> 我想建立一個提供線上課程購買和學習功能的 app，目標用戶是學生和在職人士

## 📊 輸出格式

技能輸出遵循以下結構：

```markdown
# 產品需求文件：[產品名稱]

## 1. 產品概述
[簡短描述產品目標和願景]

## 2. 目標用戶
[描述目標用戶群體和他們的特徵]

## 3. 核心功能
[列出核心功能列表]

### [功能名稱]
- [功能描述]
- [使用場景]
- [預期結果]

## 4. 非功能需求
[性能、安全、可用性等要求]

## 5. 成功指標
[如何衡量產品成功]

## 6. 實施路線圖
[開發階段和里程碑]
```
